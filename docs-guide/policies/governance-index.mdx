## Component placement — decision questions

Ask in order. First "yes" determines the category.

| Question | Category | Folder |
|---|---|---|
| Single visual atom — no children, no fetching, no arrangement? | element | `snippets/components/elements/` |
| Holds, groups, or arranges other components? | wrapper | `snippets/components/wrappers/` |
| Takes content and presents it in a formatted way? | display | `snippets/components/displays/` |
| Part of the page's outer structure, typically used once? | scaffolding | `snippets/components/scaffolding/` |
| Fetches, embeds, or binds to external/third-party data? | integrator | `snippets/components/integrators/` |

Component path: `snippets/components/<category>/<subniche>/ComponentName.jsx`

Required JSDoc: `@component`, `@type`, `@subniche`, `@status`, `@description`, `@accepts`, `@dataSource` (if integrator)

## Script placement — decision questions

### Type (Layer 1)

| Question | Type | Folder |
|---|---|---|
| Read-only scan, measure, report? | audit | `operations/scripts/audits/` |
| Produces new files from source data? | generator | `operations/scripts/generators/` |
| Enforces rules with pass/fail gate? | validator | `operations/scripts/validators/` |
| Bulk fixes or repairs existing files? | remediator | `operations/scripts/remediators/` |
| Dispatches work to other scripts or agents? | dispatch | `operations/scripts/dispatch/` |
| End-to-end automated pipeline? | automation | `operations/scripts/automations/` |

### Concern (Layer 2)

| Concern | Folder | Covers |
|---|---|---|
| content | `content/` | Docs pages, copy, SEO, veracity, quality |
| components | `components/` | Component library, registry, CSS, naming |
| governance | `governance/` | Scripts about scripts, repo structure, agent docs |
| ai | `ai/` | LLM files, agent packaging, skills sync |

Script path: `operations/scripts/<type>/<concern>/<niche>/<script-name>.js`

Required JSDoc: 11 tags in order — `@script`, `@type`, `@concern`, `@niche`, `@purpose`, `@description`, `@mode`, `@pipeline`, `@scope`, `@usage`, `@policy`
