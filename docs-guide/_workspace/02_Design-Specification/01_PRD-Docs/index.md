## **Livepeer Strategic Context**

### **Where Livepeer Is (2026)**

Livepeer is a decentralised compute network for video and AI. It started as video transcoding infrastructure and is executing the **Cascade** vision: becoming the leading platform for real-time AI video pipelines.

**The strategic shift:** From pure transcoding → real-time AI video. Over 72% of network fees are now driven by AI inference. The Cascade framework enables applications like AI-powered video agents, VTuber overlays, live video analytics, generative content, and real-time style transfer.

**The Livepeer Foundation** was established to coordinate this transition:

- **2025:** Foundation phase — establish operations, launch advisory boards, drive real-time AI capabilities
- **2026–2027:** Scaling phase — lead marketing, diversify gateway infrastructure, expand demand sources
- **2028+:** Decentralisation phase — transition governance to community

### **What Livepeer Needs (Strategic Goals)**

These are the network-wide goals that everything — including docs — must serve:

| **#** | **Strategic Goal** | **What It Means** | **Why It Matters** |
| --- | --- | --- | --- |
| **G1** | More paying users | Applications and developers paying ETH fees to use the network for video and AI compute | This is the fundamental business model. No fees = no network sustainability. |
| **G2** | More services and pipelines | More AI pipelines, BYOC containers, and compute workloads available on the network | Diversity of services attracts diversity of demand. More services = more reasons to use Livepeer. |
| **G3** | More platforms and gateways | More gateway operators running infrastructure that routes demand to the network | Gateways are the supply/demand connectors. More gateways = more diverse demand channels = healthier marketplace. |
| **G4** | More and better orchestrators | GPU operators running diverse, reliable, well-staked nodes with wide AI model coverage | Supply quality determines whether the network can fulfil the demand that G1–G3 generate. |
| **G5** | Healthy stake distribution | Active, informed delegation across high-quality orchestrators with engaged governance | Stake distribution affects network security, governance quality, and orchestrator incentives. |
| **G6** | Developer ecosystem growth | New developers, founders, and contributors entering the ecosystem through programs, grants, and community | Growth of the builder community is how G1–G5 compound over time. |

### Foundation / Network Strategic Context

Livepeer's organisational structure consists of four entities: **Livepeer Inc.** (product development, demand generation - owns Studio and Daydream), the **Livepeer Foundation** (strategic coordination, ecosystem stewardship — launched mid-2025), the **Livepeer DAO** (on-chain governance by LPT holders — funds SPEs, approves protocol upgrades), and **Special Purpose Entities (SPEs)** (mission-driven funded teams — AI Video SPE, Cloud SPE, Streamplace SPE, LiveInfra SPE).

The Foundation's strategy pipeline flows: **Advisory Boards** (Network, Growth, Governance, Capital Markets) → **Strategic Pillars** (July 2025) → **Tactical Recommendations** (July 2025) → **Workstreams** (August 2025) → **Roadmap** (roadmap.livepeer.org).

## Docs Objectives

### **What Docs Must Do (Docs Objectives)**

Docs don't exist in a vacuum. Every docs objective is derived from a strategic goal above.

| **Docs Objective** | **Serves Goal** | **What It Means Concretely** |
| --- | --- | --- |
| **D1: Convert developers to builders** | G1, G2, G6 | A developer who arrives at the docs leaves with working code. Not "I read about it" — "I built something." |
| **D2: Activate gateway operators** | G3 | Someone who wants to run a gateway can go from zero to a running, earning gateway using only these docs. |
| **D3: Activate orchestrators** | G4 | Someone with GPUs can go from zero to a registered, earning orchestrator using only these docs. |
| **D4: Activate delegators** | G5 | Someone holding LPT can delegate, monitor rewards, and understand governance using only these docs. |
| **D5: Be the canonical resource** | All | These docs are THE place to learn about Livepeer. Not Discord, not blog posts, not guesswork. If it's not in the docs, it's not documented. |
| **D6: AI-first discoverability** | G1, G6 | AI agents (Claude, ChatGPT, Perplexity) can extract correct, structured answers from these docs. Developers increasingly discover tools through AI, not search engines. |
| **D7: Low-lift maintenance** | All (sustainability) | One person can maintain the docs without knowledge loss. The system doesn't collapse when the original author leaves. |
| **D8: Clear stakeholder ownership** | All (governance) | Every section has an owner. Every page has an owner. Changes go through a defined process. |

### Docs Philosophy (How We Build)

Three principles inform how documentation is built and maintained:

1. **Documentation is infrastructure, not editorial.** It must be maintained under load, degrades without governance, and requires testing, automation, and versioning — just like code.
1. **AI is the new search.** LLMs index, summarise, and recommend products at scale. Developers begin research with AI assistants, not search engines. Semantic headings, stable URLs, machine-readable frontmatter, and llms.txt are minimum viable distribution.
1. **Agents are the new users.** Automated systems query docs to generate code, answer support tickets, and configure infrastructure without human intervention. Docs must be not just readable but executable — with explicit preconditions, verification steps, and failure modes.

### Docs Purpose

Livepeer documentation exists to take each persona from "I just heard about Livepeer" to "I am confidently doing the thing I came to do" — using only these docs and the tools they link to.

That's it. Everything else (AI discoverability, infrastructure, governance, brand) serves that purpose or it doesn't belong.

**Three jobs, in priority order:**

1. **Orient.** Answer "what is Livepeer and is it relevant to me?" for someone who just arrived.
1. **Activate.** Get each persona to their first successful action (first AI job, first gateway running, first delegation, first transcoding job).
1. **Enable.** Support ongoing operation, deeper understanding, and troubleshooting for people who are already active.

**Priority alignment with strategic goals:**

| **Priority** | **Serves** | **Rationale** |
| --- | --- | --- |
| Activate developers (AI) | G1, G2, G6 | Highest strategic value — real-time AI video is the Cascade bet. Every developer who gets their first AI job running is a potential paying user and pipeline contributor. |
| Activate gateway operators | G3 | Second priority — gateways are the demand connectors. More gateways = more diverse usage. Foundation 2026–2027 scaling phase explicitly calls out gateway diversification. |
| Activate orchestrators | G4 | Third — supply needs to meet demand. But supply without demand is wasted capacity. Orchestrator docs serve an existing, engaged community. |
| Activate delegators | G5 | Fourth — important for network health but lowest direct impact on fee generation. |
| Orient newcomers | G6 | Ongoing — the front door must work or nothing downstream works. |
