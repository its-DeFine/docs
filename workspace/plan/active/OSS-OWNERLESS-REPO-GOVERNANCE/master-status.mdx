## Purpose

Convert the Livepeer Docs repo from human-owner governance to ownerless governance — where every governed surface has one canonical source, one deterministic validator, one repair path, and one primary gate layer. No repo-tracked contract requires a named person to validate, route, repair, or merge. Six phases: foundation and baseline, bounded deterministic drift surfaces, contributor/GitHub governance, script governance schema migration, component governance schema migration, and graduation to ownerless-ready surfaces.
