### Real-time endpoint (separate path ŌĆö not REST)

`live-video-to-video` runs over the **trickle protocol** (`j0sh/http-trickle`) via WebRTC. **Not callable** via standard REST. Two access paths:
1. **Browser WebRTC** via `@muxionlabs/byoc-sdk` (see `[REVIEW]` above).
2. **Server-side trickle client** via `pytrickle.TrickleClient` or direct HTTP-trickle connection to a deployed BYOC orchestrator.

Performance: up to **25 FPS on NVIDIA RTX 4090** (CUDA 12.5, PyTorch 2.5.1+cu121).

### LLM pipeline shape

OpenAI-compatible `/v1/chat/completions` format via an Ollama-based runner. Supports Llama, Mistral, Gemma, Qwen, and other Ollama-compatible models.

### Response shapes

- Image pipelines: `{ images: [{ url, seed }] }`.
- Text pipelines (LLM, image-to-text, audio-to-text): `{ text }` or OpenAI-format completion object.

### Cold vs warm models

Warm models respond immediately. Cold models incur **30 s to several minutes** load time. Custom Hugging Face model IDs are accepted via the `model_id` parameter. LoRA models supported on relevant pipelines.

### OpenAPI spec

Generated from FastAPI routes in `ai-runner` via `gen_openapi.py` ŌåÆ `openapi_gateway.json`. Go bindings via `make ai_worker_codegen`. Rendered in v2 docs at `v2/gateways/resources/technical/api-reference/AI-API/ai`.

## ┬¦4 ŌĆö BYOC mechanics

### Container interface (PyTrickle `StreamServer`)

Every BYOC container exposes four REST endpoints automatically when wrapped by `pytrickle.StreamServer`:

| Endpoint | Method | Purpose |
|---|---|---|
| `/api/stream/start` | POST | Start a stream session. Receives `subscribe_url`, `publish_url`, `params`. |
| `/api/stream/params` | POST | Update parameters mid-stream. |
| `/api/stream/status` | GET | Check session status. |
| `/api/stream/stop` | POST | Stop current session. |

### Build workflow (six steps)

1. Implement `FrameProcessor` subclass in Python via PyTrickle. Override `process_frame(self, frame: VideoFrame) -> VideoFrame`.
2. Build a Docker container.
   - GPU base image: `nvidia/cuda:12.1.0-cudnn8-runtime-ubuntu22.04`.
   - CPU base image: `python:3.11-slim`.
3. Test locally with the `http-trickle` server.
4. Push to a container registry.
5. Deploy to the Livepeer network: `go-livepeer -byoc -byocContainerURL=<url> -byocModelID=<id>`. **`[REVIEW: confirm exact flag names against go-livepeer master ŌĆö currently flagged in `v2/developers/build/byoc.mdx`]`**.
6. Connect a client (browser via `@muxionlabs/byoc-sdk`, or server-side via direct trickle).

### Implementation source

BYOC is implemented as a `byoc/` directory inside `go-livepeer` (no standalone repo). Files:
- `byoc/byoc.go`
- `byoc/job_auth.go`
- `byoc/job_gateway.go`
- `byoc/trickle.go`
- `byoc/payment.go`
- `byoc/stream_orchestrator.go`
- `byoc/stream_gateway.go`

Production-grade as of Phase 4 (January 2026). Production workloads run via Embody SPE and Streamplace.

### Streaming-first

BYOC is **streaming-first** ŌĆö containers process frames in real-time via the trickle protocol. Batch BYOC (single request ŌåÆ single response) is not documented as a separate mode.

- `livepeer/comfystream` ŌĆö main implementation (Python).
- `yondonfu/comfystream` ŌĆö original implementation, 65 stars.
- `livepeer/ComfyUI-Stream-Pack` ŌĆö custom ComfyUI nodes for streaming, 20 stars.
- Dedicated docs: `https://docs.comfystream.org`.

### Pipeline modes

- image-to-image
- video-to-video
- audio processing
- data-channel output

### Node ecosystem (Phase 4)

- Core I/O: `LoadTensor`, `SaveTensor` (auto-converted from `LoadImage` / `SaveImage` / `PreviewImage`).
- Real-time control nodes.
- StreamDiffusion nodes.
- SuperResolution node.
- AudioTranscription nodes.

### Workflow format

Standard ComfyUI **API format** JSON. Single primary input node (`PrimaryInputLoadImage` or `LoadImage`). Single output via `PreviewImage` / `SaveImage`. Nodes auto-converted to `LoadTensor` / `SaveTensor` at runtime.

### Performance

- Up to **25 FPS** on NVIDIA RTX 4090.
- Stack: CUDA 12.5, PyTorch 2.5.1+cu121.
- Frame rate configurable 1ŌĆō60 FPS (default 24).
- First run incurs TensorRT compile cost.

### Supported models / chains

StreamDiffusion (core real-time diffusion), ControlNet (depth-guided generation), IPAdapter, FaceID, LoRAs, SuperResolution, Whisper (audio transcription), Gemma (video understanding).

### Deployment

Five paths: Docker DevContainer (pre-configured), Docker image (`livepeer/comfystream`), RunPod template (`livepeer-comfystream`), Tensordock (Python deploy script), Ansible playbook (cloud-agnostic). Pip-installable: `pip install git+https://github.com/livepeer/comfystream.git` (Miniconda + Python 3.12 + PyTorch).

A Next.js UI ships in `ui/`.

## ┬¦6 ŌĆö PyTrickle

### Repo and status

- `github.com/livepeer/pytrickle` ŌĆö created July 2025, 3 stars, 20 open issues.
- High-performance Python framework for real-time video / audio streaming over the trickle protocol (`j0sh/http-trickle`).
- The bridge between custom AI containers and the Livepeer network.

### Components

- `StreamServer` ŌĆö HTTP server with the four REST endpoints in ┬¦4.
- `FrameProcessor` ŌĆö base class for AI processors.
- `TrickleClient` ŌĆö direct client.
- `TrickleProtocol` ŌĆö high-level protocol wrapper.

### Features

- Async / await processing (asyncio).
- PyTorch tensor-based video frames (`VideoFrame`, `AudioFrame`).
- Automatic CUDA GPU memory management.
- Real-time parameter updates mid-stream.
- Configurable frame rate (1ŌĆō60 FPS).
- Audio support: mono / stereo / multi-channel.
- Built-in monitoring: FPS, latency, memory, errors.
- Auto-reconnection.
- Data flow: `Input Stream ŌåÆ Decoder ŌåÆ FrameProcessor ŌåÆ Encoder ŌåÆ Output Stream`.

### Requirements

Python 3.8+, PyTorch, FFmpeg. Install via `pip install -e .`.
