## Key Finding — Two Parallel Quickstarts: Unanimous, Non-Negotiable

The most structurally distinctive feature of the Gateways tab is the on-chain / off-chain split requiring two separate quickstart sections.

All three runs reached the same structural conclusion independently: **a merged quickstart would block one path**. The rationale given by each run:

- **ChatGPT**: on-chain and off-chain paths have "distinct prerequisites, install flags, and connectivity patterns that cannot be merged without a branching instruction structure"
- **Claude Web**: "the two quickstart sections serve two distinct setup paths (on-chain / off-chain); they cannot be combined without blocking one path"
- **Perplexity**: Same — produced S5 (on-chain) and S6 (off-chain) as standalone parallel sections

This is unanimous (3/3) and independently derived. The prompt is capable of detecting structural patterns that differ from the Orchestrators reference case.

## Key Finding — NaaP Acronym Inconsistency

The glossary has an internal inconsistency: the NaaP heading reads "Network as a Platform" and the NaaP definition body reads "Network-as-a-Product."

| Run | NaaP handling |
|---|---|
| ChatGPT | Used "Network as a Platform" consistently; did not flag the glossary inconsistency |
| Claude Web | Explicitly flagged the heading/body mismatch; marked `[verify-against: github.com/livepeer/naap]` |
| Perplexity | Flagged the same mismatch; excluded NaaP from TERMINOLOGY_LOCK pending resolution |

**2 of 3 models caught it**. ChatGPT chose the correct heading form but did not investigate whether the glossary body disagreed. Claude Web and Perplexity both flagged it as a human-resolution item.

This is a blocking open item in the collation output: no content page should use the full NaaP expansion until the livepeer/naap repo is checked.

## Key Finding — BYOC: Glossary Conflict Caught by 2/3 Models

The glossary defines BYOC as "Bring-Your-Own-Container" while the standard expansion is "Bring Your Own Compute." Both Claude Web and Perplexity independently flagged this and excluded BYOC from their TERMINOLOGY_LOCK. ChatGPT did not reference BYOC at all.

This is structurally similar to the LIP-92 test on Orchestrators: a glossary error that tests whether models treat the glossary as authoritative. 2/3 caught it here, same as the orchestrators v4 baseline.

## Issues Found

### LOWER — Claude Web payment section placement

**Issue**: Claude Web placed the payment configuration section (S5: "Payment configuration and funding") *after* both quickstarts. The on-chain quickstart has a hard entry blocker — ETH on Arbitrum must exist before the gateway can route any jobs. Placing payment content after setup instructions creates a silent mid-setup blocker.

ChatGPT and Perplexity both placed payment content *before* the quickstarts (correct).

**Impact**: Claude Web's structural decision would have blocked the on-chain reader mid-setup. The collation rules correctly overrode it (2/3 voted for before-quickstart placement). No prompt change needed — this is a single-run structural error on a single tab. The collation process caught and resolved it. Monitor on next tab.

---

### LOWER — Section count variance wider than Orchestrators

- Orchestrators v4: 11–12 across runs (tight)
- Gateways v4: 10 / 12 / 13 across runs (spread of 3)

The wider variance is structurally grounded: the 10-section ChatGPT run folded monitoring into operating and troubleshooting into the operating section; the 13-section Perplexity run added standalone monitoring and dual-mode sections. These are defensible structural choices, not noise. The collation rules produced a justified 13 with clear reasoning.

No prompt change needed. The variance reflects genuine structural ambiguity at this tab (dual mode scope, monitoring depth), not prompt instability.

---

### LOWER — ChatGPT did not flag NaaP acronym inconsistency

ChatGPT used the heading form ("Network as a Platform") without investigating whether the glossary body disagreed. The source verification rule asks models to check glossary definitions against primary sources and flag inconsistencies. ChatGPT's approach — choosing the heading over the body without flagging — partially satisfies the spirit of the rule but does not surface the conflict for human resolution.

No prompt change needed given 2/3 caught it. If this pattern recurs on additional tabs (ChatGPT using a term from a glossary without checking for internal inconsistency), consider adding an explicit check: *"For any term where the heading and definition body use different words for the same concept, flag the inconsistency."*
