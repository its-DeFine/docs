### OSS / Ownerless Repo Governance
**Folder**: [`OSS-OWNERLESS-REPO-GOVERNANCE/`](OSS-OWNERLESS-REPO-GOVERNANCE/)
**Status**: Framework defined; implementation ongoing via per-domain plans

Defines the principle that this repo has no human owner — governance is deterministic, machine-enforced, and fully documented. Every governed surface must have one canonical source, one deterministic validator, one exact repair path, and one primary gate layer. Canonical truth lives in files/generators/tests/manifests (not people). The 7 governed surfaces are: portable skills, docs-guide catalogs, UI templates, GitHub/contributor governance, scripts, components, and frontmatter.
