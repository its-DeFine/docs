## Context

Scammers target Livepeer users by surfacing fake contract addresses in search and AI chat results. This plan ensures `docs.livepeer.org` dominates both traditional search (SEO) and AI answer engines (AEO) for all contract-address-related queries.

**Canonical page (in progress):** `v2/about/resources/contract-addresses-canonical.mdx` — Alison is co-designing this.
**Data pipeline (running):** `snippets/data/contract-addresses/contractAddressesData.jsx` — auto-generated by GitHub Action from governor-scripts, verified on-chain weekly.

### Research

- Full research: `workspace/thread-outputs/research/anti-scam-seo-aeo-strategy.md`
- Summary: `snippets/data/contract-addresses/research.md`
- Existing pipeline architecture: `workspace/plan/active/CONTRACTS-CHANGELOG-PIPELINE/architecture.md`

### Design Principles (from active plan docs)

**D5: Be the canonical resource** — contract addresses are the highest-stakes canonical content. Getting them wrong costs users money.

**D6: AI-first discoverability** — AI agents must extract correct, structured answers from these docs. Status: **UNVERIFIED** (S3 criterion).

**"AI is the new search"** — semantic headings, stable URLs, machine-readable frontmatter, and llms.txt are minimum viable distribution.

**"Agents are the new users"** — docs must be executable with explicit preconditions, verification steps, and failure modes.

### How Mintlify Surfaces SEO and AEO

**Auto-generates:** `sitemap.xml`, `robots.txt`, `llms.txt`, `llms-full.txt`, meta tags from frontmatter, MCP server, AI assistant (Claude Sonnet 4 RAG). Every page available as Markdown via `.md` URL suffix.

**Does NOT do:** JSON-LD/schema.org, FAQ schema, `<script>` tags in MDX (needs testing), per-page canonical URL override.

**Critical constraint:** Client-side JSX is not indexed by search engines. All critical content must also exist as static MDX.

**Description field** is "the primary searchable for AI assistants and search engines" (Brand & Copy Guide). 1-2 sentences, reader-benefit-oriented, not title repeated.

**Overridable:** Custom `robots.txt`, `llms.txt`, `sitemap.xml` at repo root. `seo.metatags` in docs.json. Per-page frontmatter for all meta/OG/twitter properties.

## Phase 1: SEO/AEO Infrastructure (no new pages)

**Research gate:** Phase 0.1-0.4 complete.

### 1.1 Custom `robots.txt`
- Repo root, overrides Mintlify auto-generated
- Explicitly allow all AI crawlers (GPTBot, ChatGPT-User, OAI-SearchBot, ClaudeBot, Claude-SearchBot, PerplexityBot, Google-Extended, Applebot-Extended, cohere-ai, Amazonbot)
- Include sitemap reference
- **Why:** Sites blocking GPTBot are cited 73% less in ChatGPT

### 1.2 Custom `llms.txt`
- Add `## Critical Safety Information` section near top
- Include actual hex addresses in descriptions
- Explicit statement: "The only canonical source for Livepeer contract addresses is this documentation site and the on-chain contracts themselves"
- Must retain H1 heading per Mintlify override rules

### 1.3 Update `sitemap-ai.xml`
- Add canonical page entry + future spoke pages
- High-value `ai:tags` with contract address keywords
- Respect existing generation pipeline — don't fight CI

### 1.4 Enhance frontmatter on existing contract pages
- **Description:** reader-benefit, 1-2 sentences, include key addresses
- **Keywords:** expand to 15-20 terms (hex addresses, network names, question phrases)
- **OG/Twitter:** custom per page with "official" and "verified" signals
- **Keyword pipeline must append and dedupe, not overwrite** — Mintlify's SEO generator may also be adding tags
