You have GPU hardware and want to earn from the Livepeer network. The first decision is which operating path fits your situation. Hardware capability, LPT access, and operating commitment each constrain the answer differently.

Four paths exist. Three are primary entry points; the fourth applies to operators managing infrastructure at scale.

## Path comparison

| Path | LPT stake required | Hardware | Setup effort | Best for |
| --- | --- | --- | --- | --- |
| **Solo orchestrator** | Active set threshold (check [Explorer](https://explorer.livepeer.org/orchestrators)) | NVIDIA GPU with NVENC; 1-4 consumer GPUs | High (8-16 hours) | Full control over node, pricing, and earnings. Both video transcoding and AI inference. |
| **AI-focused operator** | Minimal (enough to activate) | 24 GB+ VRAM GPU | Medium (4-8 hours) | ML practitioners with high-VRAM hardware. AI routing uses capability and price, not stake rank. |
| **Pool node** | None | Any NVIDIA GPU accepted by a pool | Low (1-4 hours) | Operators with GPU but limited LPT. Earn transcoding fees without on-chain management. |
| **Pool operator / enterprise** | Active set threshold + buffer | Multiple GPUs or hosted fleet | Very high | Infrastructure operators accepting external worker GPUs and managing shared payouts. |

<Tip>
**Not sure?** Use these quick filters:

- No LPT and want to start earning now? **Pool node.**
- Have a 24 GB+ GPU but limited LPT? **AI-focused operator.** AI job routing prioritises capability and price over stake position — active set membership is not required.
- Have LPT and want direct control? **Solo orchestrator.**
- Managing 4+ GPUs or a hosted fleet? **Pool operator / enterprise.**
</Tip>

## The LPT question

LPT stake requirements vary by path. This is the primary gate that separates them.

**Solo orchestrator (video transcoding):** The node must be in the active set — the top orchestrators by total bonded stake — to receive video transcoding jobs each round. Acquiring enough LPT to compete is the primary entry barrier for this path. {/* VERIFY: current active set size and minimum stake threshold [explorer.livepeer.org] */}

**AI-focused operator:** AI inference routing is capability- and price-based. Active set membership is not required. A minimal amount of LPT (enough to activate on-chain) is sufficient. An operator with a well-configured 24 GB GPU and warm models competes from day one without substantial stake.

**Pool node:** No LPT needed. The pool operator handles staking, reward calling, and on-chain operations. Pool nodes contribute GPU capacity and receive off-chain payouts arranged with the pool operator.
