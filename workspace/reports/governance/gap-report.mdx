## P0 — Blocks production

| # | Gap | Pipeline | Impact | Fix |
|---|-----|----------|--------|-----|
| 1 | **Contract addresses workflow never dispatched on production branch.** `integrator-maintenance-update-contract-addresses.yml` exists only on `docs-v2-dev`. GitHub Actions only indexes workflows on the default branch | Data Integration | Contract data cannot auto-update on production | Cherry-pick workflow + fetch script to `docs-v2`, manual dispatch with `--dry-run` to verify |
| 2 | **OG images not in CI.** `generate-og-images.js` requires Puppeteer, runs locally only. Fallback OG is generic for all pages without a custom image | Discoverability | Social sharing shows generic image for most pages | Add headless Puppeteer CI job or pre-generate on release |

## P1 — No enforcement gate

| # | Gap | Pipeline | Impact | Fix |
|---|-----|----------|--------|-----|
| 3 | **No voice register validation.** Validators check spelling and banned words but not whether the voice matches the audience (gateway vs delegator tone) | Copy and Brand | Wrong tone can ship. Voice compliance is manual review only | Build voice-register validator that checks audience frontmatter against voice rules per page |
| 4 | **No auto-retrigger for stale data feeds.** Freshness monitor creates issues but does not re-dispatch integrator workflows | Data Integration | Stale feed data persists until a human manually re-runs the workflow | Add `repository_dispatch` from freshness monitor to the relevant integrator |
| 5 | **No content quality validation for AI files.** Validators for llms.txt and sitemap-ai.xml check existence only, not completeness or broken entries | Discoverability | Stale or incomplete AI files served to LLMs | Extend validators to check entry count, broken URLs, freshness |
| 6 | **Translation pipeline has no periodic check.** Translations are manual-dispatch only. No mechanism detects when source pages change and translations become stale | Data Integration | Translated pages drift from English source without detection | Add cron job comparing source page modification dates to translation dates |
| 7 | **No validator for script placement.** No gate enforces that `.js` files appear only in governed locations | Governance Compliance | Scripts can be added anywhere without detection | Build `check-script-locations.js` that scans for .js files outside governed paths |

## P2 — Partial enforcement (advisory or gaps)

| # | Gap | Pipeline | Impact | Fix |
|---|-----|----------|--------|-----|
| 8 | **Brand checks advisory only (P3).** Copy and voice validators do not block merge. Violations can ship to production | Copy and Brand | Style drift accumulates between manual corrections | Promote `validator-brand-check-copy-standards.yml` to P2 (required check) |
| 9 | **Component registry validation uses continue-on-error.** `generate-component-registry.js --validate-only` runs in CI but failures do not block merge | Component Health | Invalid component metadata can ship | Remove continue-on-error from the component governance sub-job |
| 10 | **Browser tests non-blocking.** `test-v2-pages.js` runs in the content quality suite but uses continue-on-error | Content Quality | Pages that fail to render can merge | Promote to blocking after known false positives are resolved |
| 11 | **Governance staleness weekly only.** Framework staleness is checked Monday 06:00 UTC. No detection between cycles | Governance Compliance | A framework could go stale mid-week | Add daily or PR-triggered staleness check |
| 12 | **JSDoc violations detected but not blocking.** `check-jsdoc-headers.js` finds 49 files with incomplete headers but the validator is not wired into a blocking workflow | Governance Compliance | Scripts with incomplete headers continue to pass CI | Wire into `validator-governance-check-governance-map.yml` as a required step |
| 13 | **49 JSDoc violations in existing scripts.** Contracts pipeline modules (8 files), language-translation libs (12 files), test files (9 files), and `.github/scripts/` (11 files with partial headers) | Governance Compliance | Existing debt undermines the standard | Triage: test files may be exempt, lib files need headers, `.github/scripts/` needs migration |

## P3 — Missing automation

| # | Gap | Pipeline | Impact | Fix |
|---|-----|----------|--------|-----|
| 14 | **No auto-repair on PR for content quality.** Remediators exist (`repair-mdx-safe-markdown.js`, `repair-page-links.js`, etc.) but no workflow auto-creates a fix PR when validators fail | Content Quality | Authors must manually identify and fix every issue | Add repair dispatch: on validator failure, run remediator and suggest PR |
| 15 | **No auto-repair on PR for brand.** `style-and-language-homogenizer-en-gb.js` requires manual dispatch | Copy and Brand | EN-GB fixes are batched instead of per-PR | Add option to run EN-GB fixer as PR comment command |
| 16 | **`.github/scripts/` migration incomplete (D-ACT-06).** 11 scripts in `.github/scripts/` should be in `operations/scripts/`. Design decision made, migration not executed | Governance Compliance | Two script locations to maintain | Execute D-ACT-06 migration: move fetch scripts to `operations/scripts/integrators/content/data/` |
| 17 | **2 snippets scripts outside governed tree.** `contractVerifierData.js` and `filterVideos.js` live in `snippets/` not `operations/scripts/` | Governance Compliance | Not covered by JSDoc validator scans | Move to governed location or register as governance exception |

## Recommendations (priority order)

1. **Cherry-pick contract addresses workflow to docs-v2** (P0, #1) — unblocks contract data automation
2. **Promote brand checks to P2** (P2, #8) — stops style drift at source
3. **Wire JSDoc validator into CI** (P2, #12) — enforces the existing standard
4. **Add freshness monitor re-dispatch** (P1, #4) — closes the stale-feed loop
5. **Build voice register validator** (P1, #3) — the biggest missing quality gate
6. **Execute D-ACT-06 migration** (P3, #16) — consolidates script locations
7. **Extend AI file validators** (P1, #5) — ensures AI surfaces are complete
8. **Add auto-repair dispatch to CI** (P3, #14, #15) — reduces manual toil
