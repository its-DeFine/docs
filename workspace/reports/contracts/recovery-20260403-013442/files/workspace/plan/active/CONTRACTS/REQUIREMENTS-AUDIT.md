## Status Key

- `correct`: current state matches the requirement
- `partial`: current state satisfies part of the requirement, but not the full contract
- `incorrect`: current state violates the requirement
- `unverifiable`: current state may be close, but the exact approved target cannot be proven from authority

## Architecture Flag Key

- `none`: no structural issue surfaced by this requirement
- `warning`: current implementation works or partly works but has structural debt
- `fail`: the current implementation violates the intended ownership, Mintlify, or repo architecture
