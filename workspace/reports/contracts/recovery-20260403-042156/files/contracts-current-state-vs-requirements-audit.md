## Highest-value current mismatches

1. The root generated schema does not match `SCH-001`.
   Current output omits a top-level `historical` key.

2. Historical reconstruction is only partially aligned to the intended truth model.
   The active run path does not call `fetchControllerSetContractInfoLogs()` or `buildHistoricalEntriesFromEventLogs()`.

3. Cadence is daily but not configurable.
   The workflow satisfies the daily default, but not the configurability requirement in `COM-023` and `REC-006`.

4. The current watched-repo surface is still four repos, not the five-repo mature target.

5. Branch anomalies do not currently block publication on their own.
   They become warnings, not failures.

## Current strengths already present

1. The repo is already GitHub-Action-driven with cron, manual dispatch, and repository dispatch.
2. The generator-owned data contract exists and writes both page-consumer exports.
3. Active/currentImplementation separation is already present in the generated registry.
4. RPC failover, explicit chain config, and controller/proxy metadata are already present.
5. The blockchain companion output already carries the required faucet wording, L2Migrator proxy/target split, and the current bans on `weak` and stale hardcoded status copy.
