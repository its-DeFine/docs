## 1. Create an account and API key

1. Go to [Livepeer Studio](https://livepeer.studio) and create an account.
2. Open the **Developers** area and click **Create API Key**.
3. Store the API key securely (e.g. in environment variables). Use it from your **backend** when calling the Livepeer Studio API.

<Warning>
 **CORS-enabled API keys** are not recommended and will be deprecated. Make requests to the Livepeer Studio API from your backend and never expose your secret API key in the browser.
</Warning>

<Info>
 Use separate accounts or [projects](/solutions/livepeer-studio/docs/reference/managing-projects) for development and production to keep environments isolated.
</Info>

## 2. Install the SDK

This example uses the JavaScript SDK and the Livepeer React library:

```bash icon="terminal"
npm install livepeer @livepeer/react
```

## 3. Set up the client

Add your API key to environment variables, then create the Livepeer client (e.g. In a backend or server context):

```tsx icon="terminal"
import Livepeer from "livepeer";

const livepeer = new Livepeer({
  apiKey: process.env.LIVEPEER_API_KEY,
});
```

## 4. Get playback info

Use the client to fetch playback info for an asset (for example, one you uploaded in the dashboard). You need the asset’s **playback ID**:

```ts icon="terminal"
import { getSrc } from "@livepeer/react/external";

const playbackId = "f5eese9wwl88k4g8"; // replace with your playback ID

export async function getPlaybackSource() {
  const playbackInfo = await livepeer.playback.get(playbackId);
  return getSrc(playbackInfo.playbackInfo);
}
```

Mintlify doesn't support `React.lazy` or dynamic imports, but `IntersectionObserver` + `useEffect` + `useRef` are all available (browser API + global React hooks). A viewport-triggered wrapper component is the cleanest solution.

The community page is heavy because it renders all at once: YouTube iframes, a GitHub README fetch, Discord data, and a Twitter iframe. A `LazyLoad` wrapper defers rendering until the section scrolls into view.

Here's what I'd build — a single reusable component:

```jsx
// LazyLoad: renders children only when scrolled into viewport
export const LazyLoad = ({ children, height = "200px", offset = "200px" }) => {
  const ref = useRef(null);
  const [visible, setVisible] = useState(false);

  useEffect(() => {
    const el = ref.current;
    if (!el) return;
    const observer = new IntersectionObserver(
      ([entry]) => { if (entry.isIntersecting) { setVisible(true); observer.disconnect(); } },
      { rootMargin: offset }
    );
    observer.observe(el);
    return () => observer.disconnect();
  }, []);

  return (
    <div ref={ref} style={{ minHeight: visible ? "auto" : height }}>
      {visible ? children : null}
    </div>
  );
};
```

Usage in community.mdx would be:

```mdx
<LazyLoad height="400px">
  <YouTubeVideoData items={youtubeDataStatic} limit={4} />
</LazyLoad>

<LazyLoad height="300px">
  <DiscordAnnouncements ... />
</LazyLoad>

<LazyLoad height="600px">
  <MarkdownEmbed url="..." />
</LazyLoad>
```

**How it works:** Each section gets a placeholder `minHeight` until it's within 200px of the viewport, then it renders. The observer fires once and disconnects — no ongoing cost.
