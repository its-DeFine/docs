There's the problem. `spawn('mint', ['dev', ...], { stdio: 'inherit' })` with no `detached` option means the child is in the same process group as the parent. When you Ctrl+C in the terminal:

1. SIGINT goes to the **entire foreground process group** (bash → node → mint)
2. The Node supervisor catches SIGINT and runs `shutdown()` which calls `stopMintChild()` → `terminateChildProcess()` → sends another SIGINT to mint
3. But mint already got the first SIGINT from the terminal and may have already started dying
4. The `shutdown()` is `async` — it awaits `stopMintChild()` with timeouts (8s SIGINT, 3s SIGTERM, then SIGKILL)
5. But `process.on('SIGINT')` handlers don't prevent Node from exiting if there's nothing keeping the event loop alive

The issue: `shutdown()` is async but the signal handler just does `void this.shutdown(signal)` — fire and forget. If Node's event loop drains before the async shutdown completes, the process exits without properly killing the child.
