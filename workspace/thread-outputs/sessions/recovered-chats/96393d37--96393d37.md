Critical review — here are the risks and gaps I see:

### Risk 1: `git rm --cached` breaks other clones
When we gitignore tracked files and run `git rm --cached`, anyone who pulls this branch will have those files **deleted from their working directory**. For a shared repo like `livepeer/docs`, this is destructive for collaborators. 

**Recommendation:** Before gitignoring anything currently tracked, confirm whether other contributors actively use docs-v2-dev. If they do, we need a different approach — perhaps a `.gitignore-local` pattern or only gitignoring items that were never tracked.

### Risk 2: Phase 1 moves break Phase 2 (docs.json)
If we `git mv` files in Phase 1 but don't update docs.json until Phase 2, the Mintlify build is broken in between. Anyone running `lpd dev` between phases gets errors.

**Recommendation:** For any file referenced in docs.json, the move and the nav update must happen in the same commit. Either: (a) do Phase 1 and 2 together per-folder, or (b) only move files NOT in docs.json during Phase 1, and handle docs.json-referenced files in Phase 2.

### Risk 3: Active work stream collisions
Other streams are actively modifying files — orchestrators veracity pass, solutions content, governance fixes. If we move or deprecate files they're working on, we create conflicts.

**Recommendation:** Before deprecating any file, check if it's in an active work stream's scope. Add a "skip — active stream" disposition option. Flag these in the backlog instead.

### Risk 4: Backlog could grow unmanageably large
With 777 MDX files plus tooling, we could easily hit 200+ backlog items. That becomes noise rather than signal.

**Recommendation:** Cap the backlog at actionable items only. Observations like "this file is fine but could be improved" don't go in the backlog. Only items that need a decision or action. Review and prune the backlog at each batch boundary.

### Risk 5: Merge may not be fast-forward
We checked local branches, but `origin/docs-v2` may have received pushes since last fetch. The plan assumes fast-forward but should have a conflict resolution protocol.

**Recommendation:** Already partially covered, but add: fetch origin/docs-v2 at the START of Phase 3, not just at merge time. If it has diverged, assess scope before proceeding.

### Risk 6: No rollback protocol
The plan creates a cleanup branch but doesn't specify what "rollback" looks like if something goes wrong mid-Phase 1.

**Recommendation:** Add explicit rollback: `git checkout docs-v2-dev` abandons the cleanup branch. If we've already merged cleanup into docs-v2-dev, we need the pre-cleanup commit SHA recorded in progress.md.

### Risk 7: CLAUDE.md duplicate entries
Lines 428-433 have duplicate key-files entries. Minor, but should be fixed as part of this cleanup — not just logged.

1. Add a "skip — active stream" disposition alongside KEEP/DEPRECATE/GITIGNORE/REVIEW-LATER
