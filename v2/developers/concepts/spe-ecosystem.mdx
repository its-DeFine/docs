<CenteredContainer style={{ width: '90%' }}>
  <Tip>Special Purpose Entities (SPEs) are the primary mechanism for community-funded work in Livepeer. Any community member can propose an SPE. Fourteen have been funded since Q1 2024, covering AI pipelines, real-time video, social video infrastructure, governance, and agent applications.</Tip>
</CenteredContainer>

<CustomDivider />

An SPE is a working group with a specific mandate, funded by an on-chain treasury vote. SPEs are accountable to the community through regular public updates and deliverable-based funding. They are the mechanism through which the Livepeer ecosystem executes work that is too focused or too fast-moving to sit inside a core team.

Each SPE sits at a specific layer of the Livepeer stack. The diagrams below show the full SPE timeline and where each entity maps to the ecosystem architecture.

For SPE voting mechanics and how to submit a proposal, see <LinkArrow href="/v2/community/livepeer-community/governance-and-foundation" label="Governance and the Foundation" newline={false} />.

## AI agents

"AI agents" in the Livepeer ecosystem refers to two distinct but complementary things: an applied project that demonstrates agent use cases at the application layer, and a developer SDK that any builder can use to construct their own agent-based applications.

### Agent SPE -- applied agent work

The Agent SPE (led by Titan Node) builds autonomous agent infrastructure powered by Livepeer's AI inference. Their primary output is a VTuber streaming system that combines:

- **ElizaOS** for agent reasoning and emotional computation
- **Unreal Engine** for real-time avatar rendering
- **Livepeer** for the video inference and streaming layer (LV2V pipeline)

SARAH, their flagship agent, is an on-chain autonomous VTuber. User interactions modify SARAH's emotional state via on-chain wallet-linked relationship data; responses are streamed live as a VTuber broadcast. The Agent-Net incentive pilot, launched June 2025, delivers over 1.5 ETH per week into the Livepeer Network through incentivised pipeline usage -- the first community-run and incentivised pipeline in the Livepeer Network.

Phase 2 (funded April 2025, 30k LPT) extended the VTuber system and built the autonomous agent infrastructure toward the broader goal of positioning Livepeer as the default decentralised compute layer for the AI agent economy.

### @livepeer/agent -- developer SDK

`@livepeer/agent` is a provider-agnostic agent runtime that ships as part of the `livepeer/storyboard` monorepo. It is a reusable developer SDK, not an SPE project.

It manages the LLM-tool loop with swappable provider backends: Gemini, Claude, OpenAI, and a unified Livepeer provider that routes all inference through the Daydream SDK (`sdk.daydream.monster`) with a single API key. The companion package `@livepeer/creative-kit` adds canvas state management, a command router, a capability resolver with automatic model fallback chains, and UI components.

<Note>
  `@livepeer/agent` and `@livepeer/creative-kit` are not yet published to npm. Use the monorepo directly at [github.com/Livepeer/storyboard](https://github.com/livepeer/storyboard).
</Note>

The relationship between the Agent SPE and `@livepeer/agent` is complementary: the SPE proves the agent use case at the application layer with real production systems; the SDK makes that same infrastructure accessible to any developer building agent-native video applications.

## Proposing an SPE

The SPE process is defined and maintained by GovWorks. The full governance guide is at [forum.livepeer.org/t/livepeer-governance-process/2767](https://forum.livepeer.org/t/livepeer-governance-process/2767).

<Steps>
  <Step title="Check existing SPEs and mandates">
    Review the SPE table above and the [SPE Dashboard](https://www.notion.so/livepeer) to confirm your proposal fills a genuine gap. Applying to an existing SPE's grants programme (such as the Transformation SPE) may be faster than a standalone treasury proposal.
  </Step>
  <Step title="Post a pre-proposal">
    Post in the [Forum Treasury category](https://forum.livepeer.org/c/treasury) using the SPE Proposal Template maintained by GovWorks. Leave at least 7 days for community discussion. Attend a Treasury Talk or Water Cooler call on Discord to gather direct feedback.
  </Step>
  <Step title="Build community support">
    Tag `@Orchestrator` in the Discord `#governance` channel to notify active validators. Surface the proposal on social channels. Quorum requires 33% of all active stake -- visibility is critical before the vote opens.
  </Step>
  <Step title="Submit on-chain">
    Submit your final proposal at [explorer.livepeer.org/treasury](https://explorer.livepeer.org/treasury). A minimum of 100 LPT staked is required to submit. GovWorks connects proposers without sufficient stake to a sponsor -- contact them via the [GovWorks Notion Hub](https://www.notion.so/livepeer).
  </Step>
  <Step title="Voting period">
    After a 21-hour delay, a 10-round voting window opens (approximately 9 days). Orchestrators and staked Delegators vote For, Against, or Abstain via the Explorer. Delegators can detach from their Orchestrator's vote and vote independently.
  </Step>
  <Step title="Result and execution">
    A proposal passes with 33% quorum and more than 50% For votes. Funds release to the specified wallet. A multi-sig is required for team proposals. Publish a transparent execution roadmap and regular update posts on the Forum from day one.
  </Step>
</Steps>
