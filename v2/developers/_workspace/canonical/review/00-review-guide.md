# Developers Audience Design — Review Guide

**Source**: Synthesised from session research + FIRST-PRINCIPLES-SITEMAP + tab analysis

## How to review

Each part is a separate file. Review in order:
- **APPROVE** — correct, move on
- **AMEND** — mostly right, note changes
- **REJECT** — fundamentally wrong, rethink

Mark verdict at top of each file.
