## How to verify an address

Every Arbitrum One contract address can be independently verified by querying the Controller on-chain. No trust in documentation is required.

<Steps>
  <Step title="Call getContract on the Controller">
    Query `getContract(keccak256("<ContractName>"))` on the Controller at `0xD8E8328501E9645d16Cf49539efC04f734606ee4` (Arbitrum One, chain 42161). The returned address is the current registered implementation.

    ```bash
    # Example: verify BondingManager proxy address
    cast call 0xD8E8328501E9645d16Cf49539efC04f734606ee4 \
      "getContract(bytes32)(address)" \
      $(cast keccak "BondingManager") \
      --rpc-url https://arb1.arbitrum.io/rpc
    ```
  </Step>
  <Step title="Cross-check against governor-scripts">
    Confirm the returned address matches the entry in [`governor-scripts/updates/addresses.js`](https://github.com/livepeer/governor-scripts/blob/master/updates/addresses.js). This file is updated by the Livepeer Foundation when governance approves an upgrade.
  </Step>
  <Step title="Inspect on Arbiscan">
    Paste the address into [Arbiscan](https://arbiscan.io) or [Blockscout](https://arbitrum.blockscout.com) to confirm contract verification status, deployment date, and transaction history.
  </Step>
</Steps>
