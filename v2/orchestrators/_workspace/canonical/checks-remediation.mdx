# Checks Remediation Companion

Companion to `checks.mdx`. For each quality check, this page identifies the best remediation path: what to run, what skill to invoke, what template to follow, or when human judgement is required.

## Remediation Types

| Type | Meaning | When to use |
|---|---|---|
| **auto** | CI script detects and/or fixes the violation | Run the script. If it has `--fix` or `--write`, it can self-heal. |
| **skill** | AI skill invocation with guided execution | Invoke the skill manually. It loads the correct rules and guides the fix. |
| **config** | Config file defines the rule; validators enforce it | Update the config source. Existing validators pick up the change. |
| **template** | Gold-standard template or section block defines the correct pattern | Copy the template structure. Replace placeholder content. |
| **manual** | Human judgement required, guided by a prompt or rubric | Follow the prompt. No script can make this decision. |

Root-cause principle: the best remediation prevents the violation at authoring time, not just catches it after the fact. Where a skill exists that loads the correct rules before writing, that is preferred over a validator that catches mistakes after.
