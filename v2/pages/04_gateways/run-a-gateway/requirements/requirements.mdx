## Gateway Set Up Requirements

<Steps>
  <Step title="Technical Knowledge" icon="brain-circuit" titleSize="h3">  
    You should have a basic understanding of system administration and CLI tooling.  
  </Step>
  <Step title="Hardware Requirements" icon="laptop-code" titleSize="h3">
    Gateways do not require GPU resources, but must be able to handle high network throughput.
    This is necessary to route requests between applications and Orchestrators.
    
    <div style={{ width: '70%', border: '1px solid #2b9a66', borderRadius: '8px', padding: '16px' }}>
    **A typical setup includes:** 
    <Frame>
      <ul style={{ listStyleType: 'disc', listStylePosition: 'inside', paddingLeft: '20px', margin: 0 }}>
        <li>4–8 CPU cores</li>
        <li>16–32 GB RAM</li>
        <li>High-speed NVMe (optional, recommended)</li>
        <li>Stable multi-region networking</li>
        <li>Linux or containerised deployment environment</li>
      </ul>
    </Frame>
    </div>
  </Step>
  <Step title="Network Requirements" icon="network-wired" titleSize="h3"> 
    Your Gateway must be reachable and responsive. 
    - Public HTTPs endpoint
    - Low-latency connectivity to Orchestrators
    - Ability to handle high request throughput

    Recommended:

    - Multi-region deployment or failover
    - Static IP or domain name (required for production)

  </Step>
  <Step title="OS & Software Requirements" icon="computer" titleSize="h3"> 
    Gateways install the [Go-Livepeer Gateway](https://github.com/livepeer/go-livepeer) Software. <br/>
    Installation requires the following OS with root or sudo access:  
    
    <div style={{ width: '100%', border: '1px solid #2b9a66', borderRadius: '8px', padding: '16px', textAlign: 'left' }}>
      <div style={{ fontWeight: 'bold', marginBottom: '12px' }}>**[OS Requirements]('')**</div>
      <div style={{ marginBottom: '12px'}}>
        <strong>Video Only Gateway:</strong> <Icon icon="linux" size={20}/>&nbsp;<Icon icon="windows" size={20} />&nbsp;<Icon icon="docker" size={20}/>
        <ul style={{ paddingLeft: '20px', margin: '4px 0' }}>
          <li>Linux, Windows or Docker containerised deployment environment</li>
        </ul>
      </div>
      <div style={{ marginBottom: '12px' }}>
        **AI or Dual Gateway:** <Icon icon="linux" size={22}/>
        <ul style={{ paddingLeft: '20px', margin: '4px 0' }}>
          <li>Linux OS (Windows and macOS support coming soon)</li>
        </ul>
      </div>
    </div>
    <br/>

  </Step>
  <Step title="Blockchain Requirements" icon="ethereum" titleSize="h3">
    For real production deployments (rather than local testing):
    
    You will need:

    <Icon icon="arrow-right"/> **An Arbitrum** [RPC](https://en.wikipedia.org/wiki/Remote_procedure_call) **URL**
      
        Popular RPC services include [Infura]("https://www.infura.io/) and [Alchemy]("https://www.alchemy.com/").
        These services provide free tiers that should be sufficient for a single Gateway.
        <span>
        If scaling beyond this, an alternative to paying for these services is to self-host an Arbitrum node.

          label="Offchain Labs Self-Hosting Guide"

    If you are running an AI Gateway, you must configure model and pipeline metadata. <br/>

  </Step>
