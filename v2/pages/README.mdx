# V2 Pages Structure

This folder contains all groupings relating to high-level TABS Layout in the Livepeer Docs.

## Hierarchy

```
00_[NameOfTopLevelTab]
    00_[NameOfGroupSection]
        00_[NameOfPageContent.mdx]
    01_[NameOfNextSection]

01_[NameOfTopLevelTab]
    00_[NameOfGroupSection]
        00_[NameOfPageContent.mdx]
    01_[NameOfNextSection]
```

Top Level Groupings contain Sections.
Sections contain page content.

## The Top Level Groupings Are

- Home
- About
- Community
- Developers
- Gateways
- Orchestrators
- Delegators

Icons come from [Font Awesome](https://fontawesome.com/icons)


## Naming & Other Conventions
- Per Tab Landing pages are called "[tab]-portal"
    - Example: "about-portal.mdx"
    - Metadata is used to add keyword "home", "index" etc, so AIs can crawl & find information.
- Quickstarts use the Livepeer custom icon

**Internal search keywords**

Enhance a specific page’s discoverability in the built-in search by providing keywords in your metadata. 
These keywords won’t appear as part of the page content or in search results, but users that search for them will be shown the page as a result.
