# Assets Folder Structure

`snippets/assets/` is in transition from the older location-based layout to the newer asset-type taxonomy.

The `/site` phase is complete:

- favicon assets now live in `logos/favicon/`
- generated OG output now lives in `media/og-images/`
- site-scoped images now live in `media/images/site/`
- canonical `sitemap-ai.xml` lives at repo root, not inside `snippets/assets/`

The `/domain` phase is not complete yet. `domain/` remains on disk while the audit in `snippets/_workspace/audit.md` determines what should move and what should be archived.

## Current Structure

```text icon="terminal" title="snippets/assets current structure"
assets/
├── README.mdx
├── decision-log.mdx
├── data/
├── domain/                   # legacy, audit-only until phase 2
├── logos/
│   ├── Arbitrum/
│   ├── favicon/              # canonical favicon bundle
│   └── products/
├── media/
│   ├── heros/
│   ├── images/
│   │   ├── showcase/
│   │   └── site/             # layered image, UK flag
│   ├── og-images/            # canonical generated OG output
│   └── videos/
└── scripts/
    └── n8n/
```

## Canonical Rules

- Do not recreate `site/`.
- Put browser favicon assets in `logos/favicon/`.
- Put generated OG assets in `media/og-images/`.
- Put site-only images in `media/images/site/`.
- Treat `domain/` as frozen pending the approved phase-two audit.
