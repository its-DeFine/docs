{/*
SECTION BLOCK: Multi-View Layout
PURPOSE: Same content structure across 2+ platform/mode variants.

WHEN TO USE:
- instruction/quickstart pages with platform variants (Docker/Linux/Windows)
- instruction/setup pages with mode variants (off-chain/on-chain)
- Any page needing 2+ dimensions of variation

WHEN NOT TO USE:
- Pages with only one variant (use plain Tabs or no tabs)
- concept, reference, resource pages (these don't have platform variants)

RULES:
- Fixed nesting hierarchy: Views (outermost) → Tabs → Steps (innermost)
- Max 3 dimensions per page — 4+ dimensions = split into separate pages
- Steps NEVER inside Accordions
- Tabs NEVER inside Steps (reverse: Steps inside Tabs)
- If steps are identical across all tabs, use single Steps block outside tabs
- Import organisation: COMPONENT → DATA → MDX PAGE → MDX COMPOSABLE

DIMENSION GUIDE:
| Dimensions | Options per dim | Pattern |
|---|---|---|
| 1 dim, 2-4 | Tabs |
| 1 dim, 5+ | Views (dropdown) |
| 2 dims | Views → Tabs |
| 3 dims | Views → Tabs → Tabs-in-steps |
| 4+ dims | SPLIT INTO SEPARATE PAGES |
*/}
