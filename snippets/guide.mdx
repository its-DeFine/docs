# Snippets Guide

This folder contains all shared resources for the Livepeer documentation: components, composables, data, templates, and assets.

---

## How Mintlify Uses Snippets

Mintlify resolves `/snippets/*` imports using root-absolute paths. Any MDX page can import from this folder.

### Import Constraints

- Use root-absolute imports for shared resources: `/snippets/components/...`, `/snippets/data/...`
- Include file extensions: `.jsx`, `.mdx`, `.json`
- JSX components cannot reliably import other JSX files or data — import data in the parent MDX instead
- If a parent MDX imports a JSX component, child MDX files also have access to it as a prop, not a component. If you need the component in both parent and child, rename the import in the child
- Relative imports (`./views/`, `./components/`) are acceptable only for page-local colocated files within one route

### Composition Pattern

Pages that are long or have multiple views (e.g. OS-specific installation tabs) are split into modular MDX files imported into a single parent page. This keeps the navigation clean while making content modular.

**Example:** The Gateways Quickstart page imports OS-specific views from colocated MDX files, shared code strings from `snippets/data/`, and gateway-specific components from `v2/gateways/quickstart/components/`.

## Theme Colours and Dark Mode

### Colour Palette

The documentation uses a consistent colour palette defined in CSS variables:

```css
:root {
  --accent: #3cb540;
  --accent-dark: #18794e;
  --hero-text: #181c18;
  --text: #717571;
  --muted-text: #9ca3af;
  --background: #ffffff;
  --card-background: #f9fafb;
  --border: #e5e7eb;
  --button-text: #ffffff;
}

.dark {
  --accent: #2b9a66;
  --accent-dark: #18794e;
  --hero-text: #e0e4e0;
  --text: #a0a4a0;
  --muted-text: #6b7280;
  --background: #0d0d0d;
  --card-background: #1a1a1a;
  --border: #333333;
  --button-text: #ffffff;
}
```

### Colour Usage

- **heroText**: Main headings and titles
- **accent**: Subtitles, highlights, brand elements, and icons
- **accentDark**: Darker accent for step icons and emphasis
- **text**: Body text and descriptions
- **mutedText**: Secondary text and code notes
- **background**: Page background
- **cardBackground**: Card and container backgrounds
- **border**: Borders and dividers
- **buttonText**: Text on buttons (typically white)

### Implementing Dark Mode

**Method 1: CSS Variables (Recommended)**

Mintlify adds a `.dark` class to the `<html>` element when dark mode is active. CSS variables automatically update based on the theme.

**Method 2: Tailwind Classes**

Only works with standard Tailwind colours. Custom hex colours require Method 1.

### Best Practices

1. Use global CSS variables — prefer `var(--accent)` over hardcoded hex
2. Test both themes — always verify components in light and dark mode
3. Provide fallbacks — use `var(--colour, #fallback)` syntax
4. Keep semantic colours fixed — trust scores (red/yellow/green), syntax highlighting, and preview callouts should not be theme-dependent

## Frame Mode

Frame mode removes all default page styling for custom layouts.

### Enabling Frame Mode

Add `mode: frame` to your MDX frontmatter.

### Frame Mode Headings

Standard markdown headings do not render in frame mode. Use custom heading components from `/snippets/components/scaffolding/frame-mode/FrameMode.jsx`.

### PageHeader Props

- `title` (string): Main heading
- `subtitle` (string, optional): Subtitle text
- `description` (string, optional): Description paragraph
- `titleColor`, `subtitleColor`, `descriptionColor` (string, optional): Override colours

Theme colours: light mode near black title (#181C18), jade subtitle (#3CB540). Dark mode near white title (#E0E4E0), dark jade subtitle (#2b9a66).
