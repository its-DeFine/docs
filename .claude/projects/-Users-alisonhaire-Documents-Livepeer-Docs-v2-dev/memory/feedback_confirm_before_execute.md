**Rule: Confirm tasks before executing. Never implement without explicit approval.**

When the user describes a problem, asks "what needs to be done", or says "we want to do X" — treat this as a request for a report or plan, NOT a go-ahead to execute.

**Why:** User said "i didn't ask you to implement anything. i just wanted a fucking report" after I executed a large batch of v2 folder renames and policy changes without being asked. Direct quote: "fucking listen and confirm tasks FIRST".

**How to apply:**
- When a user describes a problem or names a task area → produce a report/analysis of current state and what would need to happen
- Only execute when the user gives an explicit instruction: "do it", "implement this", "go ahead", "execute", etc.
- If unsure whether they want a report or execution, ask: "Do you want me to do this, or just summarize what needs doing?"
- "We want to do X" = they are describing intent, NOT giving a go-ahead
- "Let's do X" without further context = ask before acting
